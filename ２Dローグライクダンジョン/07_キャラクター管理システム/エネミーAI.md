# エネミーAI 仕様

> **関連ファイル:** `概要書.md` / `../03_ターン制_戦闘システム/グリッド移動システム.md` / `../02_ダンジョンシステム/01_環境バイオームと属性変化.md`

---

## 1. 基本設計方針

本作はターン制のグリッド移動ゲームであるため、エネミーAIも**プレイヤーが1ターン行動したときに全敵が1ターン行動する**同期型ターン制を採用する。

- 計算コストの観点から、**視野・経路探索はシンプルな実装を優先**する。
- ルールベースのステートマシンで実装し、将来的に固有パターンをオーバーライドできる構造にする。

---

## 2. エネミーの AI ステート（状態機械）

```
[待機 / Idle]
    ↓ プレイヤーが視野範囲に入る
[発見 / Alert] ← ── ── ── ──┐
    ↓ 隣接していない               │
[追跡 / Chase]                     │ プレイヤーが視野外に出る（数ターン）
    ↓ 隣接した                     │
[攻撃 / Attack] ── ── ── ── ──┘
    ↓ HP が逃走閾値を下回る
[逃走 / Flee]
    ↓ 壁に追い詰められる or 一定距離離れる
[待機 / Idle に戻る]
```

| ステート | 行動内容 |
|---|---|
| **待機 (Idle)** | ランダムな方向にふらふらと移動（1/3の確率で移動、2/3で静止） |
| **発見 (Alert)** | 1ターン動かず「!」マークを表示（プレイヤーへの警告演出） |
| **追跡 (Chase)** | A*経路探索でプレイヤーへ向かって移動する |
| **攻撃 (Attack)** | 隣接マスに移動してプレイヤーへダメージを与える（通常攻撃 or 固有スキル） |
| **逃走 (Flee)** | プレイヤーから遠ざかる方向に移動する |

## 3. 視野（感知）判定

### 3-1. 視野範囲

| エネミー種別 | 視野半径（マス） | 備考 |
|---|---|---|
| 通常モンスター | 5 | 壁は視野を遮断する |
| 徘徊型モンスター | 3 | 部屋内に留まりやすい |
| 待伏型モンスター | 隣接1マスのみ | 擬態・宝箱型など |
| ボス | 部屋全体 | 部屋に入った瞬間発見 |

### 3-2. 視線判定アルゴリズム

グリッド移動システムに合わせて **Bresenham の直線アルゴリズム** を使用する。
プレイヤーとエネミーの間のマスに1つでも壁（通行不可マス）があれば視線は通らない。

> ⚠️ 斜め方向の壁貫通をどう扱うかは `../03_ターン制_戦闘システム/グリッド移動システム.md` と整合させる必要がある。

### 3-3. 音による感知（オプション）

- プレイヤーが大きな音を発するアクション（爆弾使用・壁壊しなど）を行うと、**音源から半径8マス**以内の全エネミーが発見状態になる。
- 隠密スキルを持つジョブ（盗賊等）は感知半径を半減できる。

## 4. 経路探索（Chase 状態）

### 4-1. 採用アルゴリズム

**A\*（A スター）アルゴリズム** を採用する。

- **ヒューリスティック関数:** マンハッタン距離（`|dx| + |dy|`）
- **コスト:** 通常移動マス = 1、斜め移動マス = 1.4（斜め移動を許可する場合）
- **更新タイミング:** 自分のターンに経路を1回再計算する（毎フレームではなく毎ターン）

### 4-2. 経路ブロック時の動作

| 状況 | 動作 |
|---|---|
| 最短経路に別エネミーがいる | そのエネミーを避けて次の最短経路を探す |
| 完全に経路が塞がれている | その場で待機（攻撃範囲スキルを持つ場合は使用） |
| プレイヤーが壁の向こう | 最後に見た位置へ向かい、到達したら待機に戻る |

## 5. 攻撃パターン

### 5-1. 通常攻撃

- **隣接した 4方向（上下左右）** にプレイヤーがいるとき発動。
- ダメージ計算は `../03_ターン制_戦闘システム/シームレス戦闘.md` に準拠。
- 斜め4方向への攻撃は**デフォルトでは行わない**（一部モンスター固有スキルで解放）。

### 5-2. 固有スキル（エネミー種別ごと）

| モンスター名（例） | 固有スキル名 | 効果 | 発動条件 |
|---|---|---|---|
| スケルトンナイト | 盾殴り | 防御力を無視してダメージ | HP50%以下 |
| ゴーストアーマー | 憑依 | 1ターン無敵＋プレイヤーの位置に瞬間移動 | 3ターンごと |
| ドラゴンゾンビ | 霊炎ブレス | 直線3マスにダメージ＋霊素加速付与 | チャージ2ターン後 |
| グリモワール | 魔法複写 | プレイヤーが直前に使ったスキルをコピーして使用 | 毎ターン判定 |
| 魔物牧場脱走兵 | 暴れる | 周囲4マスにランダムダメージ＋混乱付与 | 逃走状態に入ったとき |

> ⚠️ 上記は設計案。バイオームに応じた差し替えや追加が必要。

### 5-3. チャージ型攻撃（予告演出）

一部エネミーは強力な攻撃の前に**予告アニメーション（！マークやエフェクト）を1ターン表示**する。
これによりプレイヤーに回避行動を促す。

```
[チャージターン] エネミーが光る / テキストログ「大きく息を吸い込んだ！」
  ↓ 1ターン経過
[攻撃ターン] 範囲攻撃発動
```

## 6. 逃走行動

### 6-1. 逃走発動条件（案）

| 条件 | 備考 |
|---|---|
| HP が最大HP の 25% 以下 | 全エネミー共通の基本条件 |
| 「臆病」特性のエネミー | 発見時から逃げる傾向がある |
| 捕獲対象の魔物（霊魂） | ダメージを与えると逃走しやすくなる |

### 6-2. 逃走中の行動

- プレイヤーから最も遠ざかる方向（逆方向の最短経路）に1マス移動。
- 壁に追い詰められた場合は向きを変えてランダム移動。
- 逃走状態のエネミーは**通常攻撃を行わない**（捕獲チャンスを作るため）。

## 7. エネミーの属性・タイプ分類

> 詳細なバイオームと属性相性は `../02_ダンジョンシステム/01_環境バイオームと属性変化.md` を参照。

| タイプ | AI の特徴 |
|---|---|
| **アンデッド系** | 逃走なし（HP0まで戦い続ける） |
| **魔獣系** | 逃走閾値が高め（HP50%以下で逃走も） |
| **精霊系** | 壁抜け移動が可能（通路の経路探索を無視） |
| **人型（元・霊魂）** | 話しかけると交渉・救出イベントが発生する |
| **ボス** | 固有フェーズ移行あり（後述） |

## 8. ボス AI 設計

### 8-1. フェーズ移行

ボスはHPの残量に応じてフェーズが変化し、AI行動パターンが切り替わる。

```
フェーズ1（HP 100〜61%）: 通常の追跡・攻撃
フェーズ2（HP 60〜31%）: 固有スキルの使用頻度アップ・移動速度相当の行動回数増加
フェーズ3（HP 30%以下）: エンレイジ（怒り状態）— 全能力 1.3倍・固有スキルが強化版になる
```

### 8-2. ボス固有のAI拡張

- **ザコ召喚:** 一定ターンごとに周囲にザコエネミーを召喚する。
- **領域スキル:** 部屋内に罠や障害物を新たに配置する行動。
- **行動不能耐性:** ボスには睡眠・麻痺が効かない（or 1ターンのみ有効）。

## 9. エネミーの固有データ（管理方法）

各エネミーのパラメータはデータとして外部化し、ScriptableObject（Unity）で管理する想定。

| フィールド | 型 | 説明 |
|---|---|---|
| `enemyID` | string | ユニークID |
| `enemyName` | string | 表示名 |
| `maxHP` | int | 最大HP |
| `atk` | int | 攻撃力 |
| `def` | int | 防御力 |
| `visionRange` | int | 視野半径（マス） |
| `fleeThreshold` | float | 逃走HPしきい値（0〜1） |
| `aiType` | enum | `Normal / Coward / Lurker / Boss` |
| `uniqueSkills` | List | 固有スキルのリスト |
| `dropTable` | List | ドロップアイテムとその確率 |
| `isCapturable` | bool | 捕獲・救出可能かどうか |
| `captureType` | enum | `Spirit（霊魂） / Monster（魔物）` |
