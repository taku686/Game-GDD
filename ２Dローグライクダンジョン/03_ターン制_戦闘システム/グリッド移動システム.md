# グリッド移動システム

## 概要
ゲームフィールドをマス目（グリッド）で管理し、プレイヤー・敵・アイテムがマス単位で存在・移動するシステム。  
シレン的なローグライクの根幹であり、戦闘・罠・アイテム配置など全システムの基礎となる。

---

## 1. グリッドの定義

| 項目 | 値 | 備考 |
|------|---|------|
| 1マスのサイズ | **1×1 Unity Unit** | Unityワールド座標系に準拠。ピクセル換算は不要 |
| フロアのマップサイズ | 最小: 20×20 / 最大: 40×40 Unit | フロアごとに変動可 |
| 座標軸 | X・Z軸 = 水平移動、Y軸 = 高さ | 3Dグリッドの基本構成。グリッド座標は `(gridX, 0, gridZ)` で管理 |
| セルの種類 | 床・壁・穴・水・扉・階段 | 詳細は下記 |

---

## 2. セルの種類と通行可否

| セルタイプ | プレイヤー通行 | 敵通行 | アイテム配置 | 備考 |
|-----------|-------------|-------|------------|------|
| 床（通常） | ○ | ○ | ○ | 基本マス |
| 壁 | × | × | × | 破壊不可の固定地形 |
| 破壊可能ブロック | × | × | △（破壊後に出現） | **ハンマーで破壊可能**。破壊するとアイテムやルートが出現することがある |
| 穴 | × | × | × | 落下ダメージ or 即死（要検討） |
| 水面 | △（要アイテム） | △（水棲系のみ） | × | 渡るには特殊アイテムが必要 |
| 階段 | ○（踏むと進む） | × | × | 次フロアへの移動 |
| 罠マス | ○（踏むと発動） | ○（踏むと発動） | × | 透明罠あり（要検討） |

## 3. 移動ルール

### 3-1. 基本移動

- プレイヤーは **8方向（上下左右＋斜め方向）移動を採用**。
- 斜め移動も通常移動と同じ**1ターン消費**で統一する（『シレン』準拠）。
- 斜め移動は壁の角をすり抜けない（L字コーナーは通過不可）。
  - 例：左と上側両方に壁がある場合、左斜めへの移動はキャンセルされる。

### 3-2. 壁・キャラクターとの衝突

| 衝突対象 | 処理 |
|---------|------|
| 壁 | 移動キャンセル（ターン消費なし） |
| 敵 | 自動的に攻撃判定に切り替え（ターン消費あり） |
| アイテム | **自動取得**。プレイヤーがアイテムのマスへ移動した瞬間にインベントリへ入る。ターン消費なし |

### 3-3. 自動探索モード（採用確定）

探索の負担を軽減するための自動移動機能。以下の仕様で実装する。

#### 使用条件
- **そのフロアを1度以上踏破済みであること**が前提。
  - 初見フロアでは使用不可（マップ未解放のため経路計算不能）。
  - 踏破済み判定はフロアクリア（階段踏破）時点で記録する。

#### 探索の優先順位（プレイヤーが選択可能）

| 優先モード | 行動内容 |
|-----------|---------|
| アイテム優先 | 未取得アイテムのあるマスへ最短経路で向かう。※インベントリ満杯で停止。 |
| 敵撃破優先 | 視界内または既知の敵へ向かい攻撃する。 |
| 階段優先 | 階段のマスへ最短経路で向かい次フロアへ進む。 |
| 全探索 | 未踏破マスをすべて回るよう移動する。 |

#### 罠の処理
- **発見済み・視認中の罠**：スキル等で見破っている場合、経路探索において「通行不可」として扱い、回避して移動する。
- **未発見の罠**：通常の床として扱い、通過時に踏んでしまう（プレイヤーのスキル依存）。

#### 自動停止条件
- **目的を達成した**（例：フロア内のアイテムをすべて拾い終えた、全マス踏破した等）
- **体力が一定値以下になった**（例：最大HPの20%以下。死亡によるロストを防止）
- **インベントリが満杯になった**（アイテム優先設定時）
- **未踏破のフロア（次の階）に到達した**
- **未知の敵を視認した**
- **プレイヤーが画面をタッチした**（手動キャンセル）

#### 終了時の通知
- 自動探索が完了、または上記の停止条件によって中断された際、以下の方法でプレイヤーに通知する。
  - **バイブレーション**（スマホ端末の振動）
  - **通知SE（サウンド）**の再生

## 4. 当たり判定

### 4-1. 物理的当たり判定

- 各セルは **1エンティティのみ占有可能**（重なり禁止）。
  - 例外：**プレイヤーとアイテム**、**敵とアイテム**は同マスに共存可能。
  - 敵とアイテムが同マスにある場合、**その敵を撃破してもアイテムはドロップしない**（撃破時のドロップ処理は発生させない）。
- 判定はグリッド座標の整数比較で行う（**Unityの物理エンジン・Rigidbodyは不使用**）。
- 移動はコード制御で座標を直接指定する：
  ```csharp
  transform.position = new Vector3(gridX * cellSize, 0f, gridZ * cellSize);
  ```
- 壁・床のColliderはY方向の高さを十分に確保し、キャラクターが意図せず乗り越えないよう設定する。

### 4-2. 攻撃の当たり判定

| 攻撃種別 | 判定方法 |
|---------|---------|
| 近接（隣接1マス） | 攻撃方向の1マス先のエンティティを参照 |
| 遠距離（直線） | 指定方向に向かって1マスずつ走査し最初にヒットしたものに適用 |
| 範囲（周囲Nマス） | 中心座標からN以内のマスを全列挙 |

### 4-3. 視線判定（Line of Sight）

- 敵AIがプレイヤーを「発見」するための視線判定。
- 壁・破壊可能ブロックは視線を遮る。
- 判定方針：**物理Raycast／コライダーには依存せず**、壁・破壊可能ブロックの**マス情報**に基づき、始点から終点まで直線上のマスを列挙して遮蔽を判定する（例：ブレゼンハムの直線アルゴリズム。対称性や角の扱いが必要なら別バリエーションを検討）。
- 発見時の敵の行動変化：待機 → 追跡モードに切り替え。

## 5. 入力・操作方式（タッチUI）

> **対応プラットフォーム：スマホ（iOS / Android）タッチ画面**

### 5-1. 方向入力（バーチャルコントローラー）

- 画面左下に **8方向矢印ボタン**（左上・上・右上・左・右・左下・下・右下）を配置。
- 各ボタンはグリッド状に配置（中央は「待機（ターンパス）」ボタン）。

```
  ↖  ↑  ↗
  ← [待機] →
  ↙  ↓  ↘
```

| ボタン | 動作 | 備考 |
|--------|------|------|
| 方向ボタン（各8個） | 1マス移動 or 隣接敵を攻撃 | 直接タッチで反応よく |
| 中央ボタン | 待機（1ターンパス） | HP自然回復を待つときに使用 |

### 5-2. UI詳細の扱い

- アクションボタン（インベントリ・マップ・メニュー等）の配置・機能、画面分割・ポートレイト方針、ボタンサイズ、タッチ以外の入力対応など**UI全般の仕様は、UI関連の仕様書で定義する**（本書ではグリッド・移動ロジック中心に記載する）。

## 6. カメラとスクロール

| 項目 | 内容 |
|------|------|
| カメラ方式 | プレイヤーを中心に**固定追従**。プレイヤーの操作でカメラを動かす機能は持たない（先読み防止・探索緊張感の維持） |
| 投影方式 | **Orthographic（平行投影）**推奨。2D的な見た目を維持しやすい。Perspectiveにすると奥行き感が出る（要検討） |
| カメラアングル | **斜め見下ろし（クォータービュー）**に確定 |
| スクロール方式 | マス単位でスナップ or スムーズスクロール（要検討） |
| 表示範囲 | Orthographic Sizeで調整（ピクセル解像度に依存しない） |

### 6-1. 視界・暗転ルール

フロア内のエリア種別（通路 / 部屋）によって視界範囲を変える。

| エリア種別 | 視界範囲 | 未探索マスの扱い |
|-----------|---------|---------------|
| **通路** | プレイヤーを中心に**周囲2マス**程度のみ表示 | 範囲外は暗転 |
| **部屋（フロア）** | 部屋全体を表示 | **暗転なし**（入った瞬間に全体が見える） |

- 「通路は暗く狭い・部屋に出ると視界が開ける」というメリハリを演出する。
- 視界範囲の具体値（2マスなど）は調整可能とする（要チューニング）。

## 7. 移動アニメーション

> **保留**：スライド時間・補間方式・高速移動時の挙動はいずれも、数値だけでは判断しにくく実際の動作感に依存するため、実装して動きを確認してから決定する。現時点では仕様を定めない。

## 8. フロアマップの生成との連携

- グリッドデータはダンジョン生成アルゴリズムが出力した **タイルマップ** を入力とする。
- 生成されたマップ情報をもとに各セルのタイプ・通行可否を初期化する。
- エンティティ（敵・アイテム・罠）の初期配置もグリッド上の座標で管理。
