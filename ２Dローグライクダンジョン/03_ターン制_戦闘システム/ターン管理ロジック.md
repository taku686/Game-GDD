# ターン管理ロジック

## 概要
プレイヤーと敵が交互に行動するターン制の同期システム。  
プレイヤーが何らかの「行動」を取ることで1ターンが経過し、それをトリガーに全敵キャラクターの行動処理が走る。

---

## 1. ターンの定義

| 単位 | 説明 |
|------|------|
| **1ターン** | プレイヤーが1回の有効行動を完了した時点で経過する |
| **ハーフターン** | **不採用**。速度ステータスによる中間行動は複雑性が増しプレイヤー負担が大きいため見送り |

---

## 2. ターンを消費する行動

### 2-1. プレイヤー行動と消費ターン数

| 行動 | ターン消費 | 備考 |
|------|-----------|------|
| 移動（1マス） | 1 | **8方向移動（斜め含む）採用。スマホタッチ操作の8方向ボタンで入力** |
| 通常攻撃 | 1 | 隣接マスの敵を攻撃 |
| アイテム使用 | 1 | 消費/投擲問わず |
| 待機 | 1 | 何もしない（ターンパス） |
| 罠踏み | 1 | 罠発動も1ターン消費として扱う |

### 2-2. ターンを消費しない操作

| 操作 | 備考 |
|------|------|
| アイテム拾得（自動） | プレイヤーがアイテムのマスに移動した瞬間に自動取得。ターン消費なし |
| アイテム装備・変更 | インベントリからの装備切り替えはターン消費なし |
| メニューを開く/閉じる | ゲーム時間は停止 |
| インベントリ確認 | ゲーム時間は停止 |
| マップ確認 | ゲーム時間は停止 |

---

## 3. ターン処理フロー

```
[入力待ち（プレイヤーターン）]
    ↓ プレイヤーが有効行動を選択
[プレイヤーの行動処理]
    ↓ 攻撃・移動・アイテム使用などを解決
[ターン経過イベント処理]
    ↓ HP自然回復・状態異常カウント・罠起動など
[全敵キャラクターのAI処理（敵ターン）]
    ↓ 各敵の行動を順次解決
[フロアイベント処理]
    ↓ ダンジョン特殊イベント・エネミースポーン確認など
[入力待ち（プレイヤーターン）] ← ループ
```

---

## 4. 敵ターンの処理順序

> **決定：敵の行動順は「スピード（速度ステータス）依存」とする。**

- 各ターン開始時、フロア上の全敵を **スピード値の高い順** にソートして順番に行動させる。
- スピードが同値の場合は **出現順（ID順）** を優先。
- プレイヤーが既にその敵を倒している場合はスキップ。

### スピードステータスの定義

| スピード値 | 意味 | 例 |
|---------|------|---|
| 高い | そのターン、より早く行動する | 素早い敵・ボス |
| 低い | そのターン、より遅く行動する | 重装甲の敵 |
| 同値 | ID順で解決 | 通常モブ多数 |

### プレイヤーのスピードとの関係

- プレイヤー自身もスピード値を持つ。
- プレイヤーは常に先行して行動する（プレイヤーターン優先）。
- スピードはあくまで **敵同士の行動順** を決定するパラメータとして使用する。
- 装備・スキル・アイテムで敵のスピードを下げる（鈍足デバフなど）ことで有利を作れる。

## 5. 状態異常とターン管理

### 設計方針

- **同時付与は1つまで**（複数重複で詰み状態を防ぐ）
- 各状態異常には必ず**対策アイテム or 耐性装備**を用意する（→装備強化・町育成の動機に繋げる）
- 持続時間は**短め（1〜5ターン程度）**を基本とし、長引く不快感を防ぐ

### 採用状態異常一覧

| 状態異常 | ターン単位の挙動 | 解除条件 |
|---------|----------------|---------|
| 毒 | 毎ターン終了時にHP減少 | 一定ターン経過 / 解毒アイテム |
| 混乱 | 移動・攻撃方向がランダムになる | 一定ターン経過 |
| 睡眠 | 行動不能。攻撃を受けると解除 | 攻撃を受ける / 一定ターン経過 |
| 麻痺 | 行動不能（カウントダウン） | 一定ターン経過 / 回復アイテム |
| **凍結（氷）** | **移動・攻撃・アイテム使用・装備変更がすべて不能。攻撃を受けても解除されない** | **一定ターン経過のみ（推奨：1〜2ターン）** |
| **火傷** | **毎ターンHP減少 ＋ 与える物理ダメージが半減** | 一定ターン経過 / 回復アイテム |
| 強化バフ（攻撃力アップ等） | 一定ターン後に自動解除 | 一定ターン経過 |

### 追加候補（要検討）

| 状態異常 | 効果案 | 採用する場合のメリット |
|---------|-------|-------------------|
| 呪い | 装備を外せなくなる | 装備管理ゲームとの相性◎ |
| 鈍足 | 対象のスピード値を下げる（敵の行動順が後回し） | 既存のスピードシステムと自然に連携 |
| 弱体化 | 防御力半減（一定ターン） | 毒・火傷と差別化しやすい |
| 暗闇 | 攻撃命中率ダウン / 視野が狭まる | 探索の緊張感が増す |

---

## 6. ターン経過による環境変化

| 要素 | 変化タイミング | 内容 |
|------|-------------|------|
| 空腹度 | 一定ターンごと | 空腹度が減少。0になると毎ターンHP減少 |
| 敵スポーン | **フロア移動時に確定** | フロア生成時に敵の種類・数・配置をすべて決定。ターン中の追加スポーンなし |
| アイテム生成 | **フロア移動時に確定** | フロア生成時にアイテムの種類・数・配置をすべて決定。ターン中の追加生成なし |
| 罠 | プレイヤー・敵が踏んだとき | 即時発動 |
